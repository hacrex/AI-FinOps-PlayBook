# Contributing to AI FinOps Playbook

Thank you for contributing! This playbook grows through real engineering experience — your production stories, tool comparisons, and worked examples make it better for everyone.

---

## Ways to contribute

- **Fix a doc** — typos, outdated tool versions, broken links
- **Improve an example** — add real numbers, a better scenario, or a code snippet
- **Add a tool** — something missing from the stacks
- **Write a new technique** — open an issue first to align on scope
- **Share a lab exercise** — hands-on configs go in `labs/`

## Good first issues

Look for issues labelled `good-first-issue` — these are well-scoped and don't require deep prior context.

## Doc format

Each technique doc follows this structure. Please keep it consistent:

```markdown
# Technique name

> One-line summary

## What it is
## Why it matters
## How it works
## Tools
## Example
## Further reading
```

## Pull request checklist

- [ ] Follows the doc format above
- [ ] No broken links
- [ ] Spell-checked
- [ ] Added to the README table of contents if it's a new doc

## Code of conduct

Be direct, be kind, be constructive. Engineering disagreements are great — personal attacks are not.
